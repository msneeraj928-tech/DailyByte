## Day 11-02-2025: SpOoKy~CaSe

### Difficulty: Easy

## Problem Statement

You are given a string representing a variable name. Convert it to "spooky case" using the following rules:

- Replace all underscores (`_`) and hyphens (`-`) with a tilde (`~`).
- Capitalize the first letter of the string, and then alternate case for each subsequent character (uppercase, lowercase, uppercase, ...). When counting positions for alternating case, ignore the tilde characters â€” they should remain `~` in the output and not affect the alternation count.
- Non-letter characters (digits, punctuation) should remain unchanged, but they do count toward the alternation (unless they are `~`).
- Preserve the order of characters; only change separators and letter casing.

Return the transformed string.

Input: "hello_world"
Output: "HeLlO~wOrLd"

```

  v -> V, a -> a, r -> R, 1 -> 1 (non-letter but counts), n -> N, a -> a, m -> M, e -> e
- Result â†’ "VaR~1~NaMe"
```

## Constraints

- 1 <= s.length <= 10^4
- s consists of printable ASCII characters.
- The string may contain letters, digits, underscores (`_`), hyphens (`-`), and other punctuation.
- Treat `~` as a literal separator if already present; do not replace `~` (but do ignore it when counting positions).

## Suggested Approach (short contract)

- Input: a string `s`.
- Output: a new string where `_` and `-` are replaced with `~`, and non-`~` characters are alternately cased starting with uppercase.
- Error modes: if `s` is empty, return an empty string.
- Success: produce the correctly transformed string.

Edge cases:
- Empty string.
- Strings containing only separators (e.g., "___" â†’ "~~~").
- Strings with digits/punctuation (they remain unchanged but count).
- Strings that already contain `~` characters.

## Time & Space Complexity

- Time: O(n) â€” single pass to transform characters.
- Space: O(n) â€” output string of same length as input (O(1) extra working space if allowed to modify in-place on a mutable array).

## Example Implementation Ideas

- Iterate once, build an output list/array of chars; for each char:
  - If char is `_` or `-`, append `~` and continue (do not flip the case counter).
  - Else if char is `~`, append `~` (do not flip the counter).
  - Else (non-`~`), decide upper/lower based on the current state; if char is a letter, change case; otherwise append as-is. After appending a non-`~` char (letter or not), flip the state.
