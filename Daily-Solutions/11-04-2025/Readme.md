## ğŸ·ï¸ File Naming Convention

### Required Format
```
githubusername_ERPID.extension
```

### Components

1. **GitHub Username**
   - Your GitHub username in lowercase
   - Use underscores or hyphens if part of your username
   - No spaces allowed

2. **Underscore Separator**
   - Single underscore `_` between username and ERP ID
   - **Required separator**

3. **ERP ID**
   - Your institution ERP/Student ID
   - Numeric identifier
   - No special formatting needed

4. **File Extension**
   - Language-appropriate extension
   - Common: `.py`, `.js`, `.java`, `.cpp`, `.c`, `.go`, `.rs`

### âœ… Valid Examples

```
sayandev731_2024001.py
john-doe_2024002.js
alice_smith_2024003.java
bob_jones_2024004.cpp
charlie_2024005.go
```

### âŒ Invalid Examples

```
solution.py                    # Missing username and ERP ID
john_solution.py              # Missing ERP ID
2024001.py                    # Missing username
john doe_2024001.py           # Spaces not allowed
JOHNDOE_2024001.py            # Should be lowercase
johndoe-2024001.py            # Use underscore, not hyphen between username and ERP
```

## ğŸ’¡ Tips for Contributors

### Writing Quality Solutions

1. **Clear Variable Names**
   ```python
   # Good
   uppercase_count = 0
   current_char = string[i]
   
   # Bad
   cnt = 0
   c = s[i]
   ```

2. **Helpful Comments**
   ```python
   # Good
   # Skip tilde characters when counting position
   if char == '~':
       result.append(char)
       continue
   
   # Bad
   if char == '~':  # check
       result.append(char)
       continue
   ```

3. **Test Cases**
   ```python
   # Include test cases that verify correctness
   def test():
       assert solution("input") == "expected"
       assert solution("edge_case") == "expected"
   ```

4. **Complexity Analysis**
   ```python
   """
   Time Complexity: O(n) - single pass through string
   Space Complexity: O(n) - result string of same length
   """
   ```
